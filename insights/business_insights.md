# AI Spend Intelligence — Business Insights Report
**Project:** AI Spend Intelligence & Optimization Dashboard  
**Period:** January 2024 – December 2024  
**Dataset:** 5,000 AI usage records · 8 teams · 6 tools · 75 users  

---

## Executive Summary

This report presents key findings from analyzing 12 months of AI tool usage
across the organization. Total AI spend in 2024 was **$220.20** across
5,000 requests from 75 active users in 8 teams. Our analysis identified
**$142.32 in wasteful anomalous spend** and a **8.0% retry rate** that
represents additional rework cost.

---

## Key Findings

### Finding 1 — Spend Concentration (SQL Query 2)
**The Engineering team drives the majority of AI spend.**

- Engineering accounts for the largest share of total spend
- Primary tools: GitHub Copilot + ChatGPT-4 for code generation and bug fixing
- Per-user spend in Engineering is significantly higher than other teams
- **Recommendation:** Introduce a tool selection policy — route simple queries
  to cheaper models (GPT-3.5, Gemini) and reserve GPT-4 for complex tasks only

### Finding 2 — Wasteful Anomalous Requests (Python NumPy + SQL Query 7)
**388 requests were flagged as anomalous using Z-Score and IQR methods.**

- These requests consumed 5–12× more tokens than the team average
- Root cause: Users sending entire documents/files to AI instead of summaries
- Estimated waste: $142.32 in 2024
- **Recommendation:** Set per-request token limit of 4,000 tokens at API level

### Finding 3 — High Retry Rate (SQL Query 9)
**8.0% of all requests are retried due to poor response quality.**

- Data & Analytics team has the highest retry rate (10.3%)
- Retry = user had to re-ask because the first response was unsatisfactory
- This represents direct wasted spend (double cost for same task)
- **Recommendation:** Run prompt engineering workshop targeting top retry teams

### Finding 4 — Tool ROI Mismatch (SQL Queries 3 and 5)
**Expensive tools are used for simple tasks that cheaper tools can handle.**

- ChatGPT-4 is used for Email Drafting (avg quality 3.8/5)
- ChatGPT-3.5 delivers nearly identical quality (3.7/5) at 70% lower cost
- Gemini-Pro delivers competitive quality for Customer Queries
- **Recommendation:** Create a tool routing decision matrix by use case complexity

### Finding 5 — Peak Usage Windows (SQL Query 10)
**AI usage peaks at 9–11am and 2–4pm (core work hours).**

- Usage drops sharply after 6pm — healthy work-life pattern
- Late evening anomalies are worth investigating separately
- **Recommendation:** No action needed — usage pattern is aligned with work hours

### Finding 6 — Budget Tracking (SQL Query 8)
**Budget vs actual analysis reveals some teams exceed monthly allocations.**

- Teams with high anomaly counts are most likely to breach budget
- Current system has no automated early warning mechanism
- **Recommendation:** Set automated Slack/email alert at 80% budget utilization

## Interview Talking Points

**"Tell me about this project in 60 seconds:"**

*"I built an end-to-end AI cost monitoring dashboard that tracks how a company
spends money on AI tools like ChatGPT and Copilot. I analyzed 5,000 usage events
across 8 teams using 15 SQL queries — including CTEs, window functions like LAG
and RANK OVER PARTITION, and conditional aggregations. I used Python with pandas
for data cleaning, NumPy for statistical anomaly detection using Z-Score and IQR
methods, and Matplotlib for visualizations. The dashboard is auto-generated by
Python — which means the numbers update automatically when data changes. Key
finding: I identified $142 in anomalous waste from 388 flagged requests, and
recommended a tool routing policy that could save 40–60% on per-request costs
for low-complexity tasks."*
